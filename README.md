# Описание системы формирования изображения

## 1. Источник света

Рассматривается удалённый источник света, у которого все фотоны распространяются параллельно. Источник характеризуется интенсивностью — количеством испускаемых фотонов. Каждый фотон имеет характеристику длины волны.

## 2 Объект

Объект имеет поверхность с определёнными характеристиками. Ключевое свойство объекта — способность отражать и поглощать фотоны.

## 3 Матрица камеры

Матрица состоит из фотодиодов и воспринимает световое излучение. В зависимости от длины волны у фотодиодов изменяется сопротивление. Именно на основе этого изменения и производится считывание данных.

## 4 Оцифровка сигнала

Напряжение с матрицы поступает на пины микроконтроллера. Аналоговый сигнал оцифровывается через АЦП и преобразуется в значение от 0 до максимального значения разрядности АЦП (например, до 1024 и выше).

## 5 Формирование изображения

Из цифрового сигнала необходимо получить итоговое изображение в нужном разрешении и вывести его пользователю.

# Задачи для разработки симуляции камеры

## Калибровка матрицы
Настройка системы таким образом, чтобы при различных условиях освещения (яркость, интенсивность источника) изображение формировалось корректно, без так называемых «битых пикселей».  
## Моделирование источника света

Задать параметры источника: интенсивность, длина волны  
Реализовать модель параллельного пучка фотонов

## Моделирование объекта

Реализовать модель поверхности объекта  
Задать характеристики отражения и поглощения фотонов в зависимости от длины волны

## Моделирование матрицы фотодиодов

Реализовать матрицу фотодиодов с заданным разрешением  
Смоделировать изменение сопротивления фотодиода в зависимости от длины волны и интенсивности падающего излучения  
Реализовать считывание значений с каждого пикселя матрицы

## Формирование изображения

Разработать алгоритм сборки изображения из значений матрицы  
Определить разрешение и формат итогового изображения  
Реализовать вывод изображения пользователю

## Калибровка матрицы

Разработать алгоритм калибровки при различных условиях освещения  
Реализовать компенсацию битых пикселей  
Проверить корректность работы при разных значениях яркости и интенсивности источника
Стек технологий для симуляции камеры

Архитектура
Python Backend (симуляция) ←→ Streamlit (GUI + визуализация)

Backend — Python
Симуляция

NumPy — матрицы пикселей, расчёт интенсивности, моделирование фотонов
SciPy — оптические расчёты, свёртки, физические модели
OpenCV — обработка изображений, калибровка, коррекция битых пикселей


Frontend — Streamlit
Визуализация

st.image — вывод изображения с матрицы в реальном времени
st.empty — обновление кадра без перезагрузки страницы

UI / элементы управления

st.slider — яркость, длина волны, интенсивность источника
st.sidebar — панель параметров симуляции

Графики

st.line_chart / st.bar_chart — гистограммы интенсивности, спектральные графики
Matplotlib / Plotly — продвинутые графики, встроенные через st.pyplot / st.plotly_chart


Поток данных
Параметры (st.slider) → NumPy симуляция → OpenCV обработка
                                                  ↓
                                         st.image (вывод кадра)

Итоговый стек
