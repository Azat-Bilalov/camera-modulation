## 2. Общая архитектура

Симулятор представляет собой **последовательный пайплайн** из четырех стадий:

```
Scene Source -> Optics -> Sensor & ADC -> Visualization
    (3D)         (3D)        (3D->2D)       (RGB image)
```

Каждая стадия общается со следующей через **строго типизированные контракты** — `dataclass` из `workspace/models.py`. Это исключает рассогласование форматов данных между разработчиками.

## 3. Уровни пайплайна

### 3.1. Scene Source — модель сцены и источника

**Участник:** Гостев Максим  
**Файл:** `workspace/scene_source/scene_models.py`

#### Что происходит

Строится дискретная спектральная карта сцены размером `H x W` пикселей. Для каждой точки плоскости вычисляется собственный спектр с учетом:

- **Позиции точечного источника** `[x, y, z]`
- **Коэффициента отражения** объекта `reflectance(λ)` — загружается из CSV спектрофотометра
- **Геометрического затухания** по закону обратных квадратов
- **Косинуса угла падения** (Lambert cosine law)

#### Формулы

Для каждой точки объекта `(x, y, 0)`:

```
r = sqrt((x - x_src)^2 + (y - y_src)^2 + z_src^2)
cos(theta) = z_src / r
I_point(lambda) = I_source(lambda) * cos(theta) / r^2 * reflectance(lambda)
```

где `r` — расстояние от источника до точки, `cos(theta)` — косинус угла между нормалью плоскости и лучом света.

#### Входные данные

- `radiation` — спектр источника (единицы, 36 точек 380–730 нм, шаг 10 нм)
- `source_xyz` — координаты источника `[x, y, z]`
- `reflectance` — спектральный коэффициент отражения объекта (нормирован к `[0, 1]`)
- `object_width`, `object_height` — размер сцены в пикселях
- `point_size` — физический шаг сетки (мм)

#### Выходные данные

- `SpectralImage` — 3D-массив `(H, W, 36 bands)`, каждый пиксель содержит полный спектр

### 3.2. Optics — оптический тракт

**Участник:** Андронаки Егор  
**Файл:** `workspace/optics/optics_transformer.py`

#### Что происходит

Спектральная карта `(H, W, 36 bands)` разбивается на **три оптических канала** по длинам волн:

| Канал | Диапазон | Индексы спектра |
|-------|----------|-----------------|
| **B** (Blue) | 380–480 нм | `380 <= λ < 480` |
| **G** (Green) | 480–600 нм | `480 <= λ < 600` |
| **R** (Red) | 600–730 нм | `600 <= λ <= 730` |

Для каждого канала энергия суммируется по соответствующим длинам волн и умножается на коэффициент пропускания оптики.

#### Формулы

```
R[x,y] = sum( I[x,y,λ] for λ in 600..730 ) * transmission_R
G[x,y] = sum( I[x,y,λ] for λ in 480..600 ) * transmission_G
B[x,y] = sum( I[x,y,λ] for λ in 380..480 ) * transmission_B
```

#### Входные данные

- `SpectralImage` — 3D-спектральная карта сцены
- `OpticsConfig` — конфигурация с `transmission = [R, G, B]`

#### Выходные данные

- `SensorExposure` с полем `channel_irradiance: (H, W, 3)` — карта освещенности на сенсоре до электроники

### 3.3. Sensor & ADC — сенсор и аналого-цифровое преобразование

**Участник:** Литвиненко Эрнест  
**Файл:** `workspace/sensor_adc/sensor_pipeline.py`

#### Что происходит

Каждый из трех каналов `(H, W)` проходит через физическую модель фотоматрицы независимо:

1. **Накопление заряда:** входная экспозиция умножается на `gain` и добавляется темновой сигнал `dark_offset`
2. **Квантование АЦП:** аналоговый заряд преобразуется в целочисленный код с заданной разрядностью

#### Формулы

```
Q[x,y,c] = max(0, E[x,y,c] * gain + dark_offset + FPN[x,y])
```

gде `FPN[x,y]` — фиксированный паттерн-шум (`((x*13 + y*7) % 11) / 5000`).

```
D[x,y,c] = round( clamp(Q[x,y,c] / full_scale, 0, 1) * (2^bit_depth - 1) )
```

#### Входные данные

- `SensorExposure.channel_irradiance` — `(H, W, 3)` float
- `SensorConfig` — `gain`, `dark_offset`, разрешение
- `AdcConfig` — `bit_depth`, `full_scale`

#### Выходные данные

- `rgb_int: (H, W, 3)` — цифровой RGB-кадр после АЦП

### 3.4. Visualization — визуализация и экспорт

**Участник:** Лиличев Егор  
**Файл:** `workspace/visualization/visualization.py`

#### Что происходит

- **Нормализация** RGB-кадра в диапазон `0..255` с гамма-коррекцией (`gamma = 0.4`)
- **Сохранение** в BMP 24-bit через `save_rgb_bmp`
- **Верификация** — проверка диапазона АЦП, клиппинга, статистики по кадру
- **Генерация отчета** с полными параметрами прогона

#### Подход к нормализации

```
V_norm = V / V_max                         # деление на глобальный максимум
V_display = (V_norm ^ gamma) * 255         # гамма-коррекция для визуализации
```

Гамма `< 1` поднимает тени и делает слабый сигнал видимым (важно после геометрического затухания `1/r^2`).

#### Входные данные

- `rgb_int` — `(H, W, 3)` int от АЦП
- `PipelineArtifacts` — полный набор промежуточных данных

#### Выходные данные

- `final_image.bmp` — 24-битное RGB-изображение
- `workspace_report.txt` — текстовый отчет

## 5. Подход к разработке

### 5.1. Ролевая изоляция

- Каждый разработчик работает **только в своей подпапке**
- Нет прямых импортов между `scene_source/` и `sensor_adc/` — только через `workspace/models.py`
- Архитектор (`main.py`) собирает все модули воедино

### 5.2. Контрактное программирование

- Вход и выход каждой функции описан через `dataclass`
- Типы проверяются линтером (`ruff`)
- Несовпадение размерностей (например, 1D vs 3D) выявляется на этапе сборки

### 5.3. Единый источник данных

- Спектральная ось **одна** для всего пайплайна
- Отражение объекта загружается из **CSV спектрофотометра** и нормируется
- Геометрия источника задается единожды и прокидывается на все уровни

### 5.4. Интерактивный интерфейс

Помимо CLI (`workspace/main.py`) реализован **Streamlit UI** (`streamlit_app/app.py`), позволяющий:

- Загружать собственные спектры отражения
- Редактировать позицию источника и размер сцены
- Настраивать gain, full scale, пропускание оптики
- Переключаться между автонормализацией и физическими значениями АЦП
- Визуализировать 3D-геометрию источник–объект

## 7. Ключевые особенности реализации

| Фича | Описание |
|------|----------|
| **Геометрическое затухание** | Учитывается `cos(theta) / r^2` для каждого пикселя индивидуально |
| **RGB-разделение** | Спектр делится на 3 канала по физическим диапазонам длин волн |
| **Без Bayer CFA** | Каждый канал проходит через сенсор и АЦП независимо, сохраняя полное разрешение |
| **Гамма-коррекция** | `gamma = 0.4` компенсирует слабый сигнал при удалении источника |
| **Физический режим** | Возможность отключить нормализацию и видеть реальную яркость АЦП |
| **Реальные спектры** | Поддержка CSV от спектрофотометра (колонка `value`, 380–730 нм) |
