# Архитектура

## Базовый pipeline

Рекомендуемый pipeline:

`Source -> SceneSpectralResponse -> OpticalEncoder -> SensorModel -> ADC -> Reconstruction -> Export`

## Слои системы

### 1. Сцена и излучение

Отвечает за формирование спектральной карты сцены.

Вход:

- параметры источника света;
- геометрия или изображение объекта;
- спектральные коэффициенты отражения или поглощения объекта.

Выход:

- `SpectralImage(H, W, L)`.

### 2. Оптический тракт

Отвечает за преобразование спектральной карты в распределение энергии на сенсоре.

На этом слое моделируются:

- разделительные призмы;
- отражающие элементы;
- кодирующие маски;
- сведение каналов;
- спектральные потери оптики.

Выход:

- `SensorExposure(Hs, Ws, L)` или уже суммарная экспозиция по каналам.

### 3. Сенсор и электроника

Отвечает за формирование цифрового сигнала.

На этом слое моделируются:

- квантовая эффективность матрицы;
- накопление заряда;
- шумы, дефектные пиксели, насыщение;
- АЦП;
- калибровка.

Выход:

- `DigitalFrame(Hs, Ws)`.

### 4. Реконструкция и экспорт

Отвечает за сборку итогового изображения и его сохранение.

На этом слое выполняются:

- нормализация;
- коррекция дефектов;
- преобразование в целевой формат;
- сохранение `bmp` и служебных массивов.

## Рекомендуемое разбиение на модули

- `simulation/source.py`
- `simulation/object_model.py`
- `simulation/spectral_scene.py`
- `simulation/optics.py`
- `simulation/sensor.py`
- `simulation/adc.py`
- `simulation/reconstruction.py`
- `simulation/export.py`
- `simulation/calibration.py`

## Рекомендуемые публичные функции

- `simulate_scene(source_cfg, object_cfg) -> SpectralImage`
- `simulate_optics(scene, optics_cfg) -> SensorExposure`
- `simulate_sensor(exposure, sensor_cfg) -> ChargeMatrix`
- `digitize(charge, adc_cfg) -> DigitalFrame`
- `reconstruct(frame, recon_cfg) -> ImageFrame`
- `export_image(image, path, format="bmp") -> None`

## Почему это лучше прямого копирования схемы по блокам

- Архитектура сохраняет физический смысл.
- Оптические детали можно усложнять постепенно.
- Сенсор можно тестировать отдельно от сцены.
- Реальный спектрометр можно подключить как альтернативный источник спектральных данных.

## Главный принцип

Схема на рисунке важна как физический ориентир, но код должен строиться вокруг устойчивых данных и интерфейсов, а не вокруг номеров блоков на иллюстрации.
