# Интеграция со спектрометром

## Зачем это учитывать уже сейчас

Если преподаватель захочет сопоставить симуляцию с реальным спектрометром, архитектура должна позволять подменять синтетические спектральные данные измеренными.

Поэтому симулятор должен быть готов к двум режимам:

- полностью синтетический режим;
- гибридный режим, где спектр источника или объекта приходит из реального прибора.

## Что именно может поставлять спектрометр

- спектр источника света;
- спектр отраженного сигнала от объекта;
- калибровочные эталоны;
- контрольные точки для проверки симуляции.

## Где вставлять реальные данные

### Вариант 1. Подмена модели источника

Вместо искусственно заданного спектра используется измеренный спектр источника:

`MeasuredSpectrum -> SourceConfig`

### Вариант 2. Подмена спектрального отклика объекта

Вместо аналитической функции отражения используется измеренная кривая отражения или пропускания.

### Вариант 3. Верификация симуляции

Сначала симуляция строит спектральную карту, затем результаты сравниваются с измерениями по выбранным длинам волн.

## Что нужно заложить в интерфейсы

- единый формат спектральной оси;
- модуль импорта спектральных данных;
- нормализацию единиц измерения;
- ресемплинг спектров к сетке симуляции;
- сохранение эталонных и измеренных данных рядом с результатами эксперимента.

## Практические требования

- спектры должны храниться отдельно от кода;
- для каждого эксперимента нужен файл конфигурации;
- должна быть функция приведения измеренного спектра к внутреннему формату;
- должно быть возможно прогнать симуляцию как на синтетическом, так и на измеренном спектре без переписывания остального кода.

## Минимальный контракт для импорта спектра

Пример логического интерфейса:

- `load_spectrum(path) -> wavelengths_nm, values`
- `resample_spectrum(wavelengths_nm, values, target_axis) -> np.ndarray`
- `normalize_spectrum(values, mode="energy") -> np.ndarray`

## Риски

- у прибора и симулятора могут быть разные спектральные сетки;
- реальные данные будут шумными;
- могут отличаться единицы измерения;
- часть длин волн может отсутствовать;
- без нормализации сравнение будет некорректным.

## Вывод

Совместимость со спектрометром достигается не добавлением отдельного блока в конце, а тем, что спектральные данные становятся стандартным входом для всей модели.
